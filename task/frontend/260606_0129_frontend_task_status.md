## 문제
기존 MiscLogsSection 은 **모든 파일의 모든 행을 한 번에 DOM 렌더** + 매 키 입력마다 전 행 검색 →
파일/행 수가 늘자 랙·스크롤 지옥.

## 변경 (프론트 전용, 백엔드 무변경)
마스터-디테일 + 클라이언트 페이지네이션으로 재작성:
- **좌(파일 선택기)**: 행 수 내림차순 정렬, 파일명 검색, 행 수 배지, 선택 하이라이트. 한 번에 1파일만 선택.
- **우(상세)**: 선택 파일 행만 처리 — 파일 내 값 검색 + **페이지네이션 50행/페이지**(prev/next·범위 표시).
  보이는 페이지(≤50행)만 DOM 에. 시각형 필드(time/date 키 + 숫자)는 `prettyValue` 로 사람이 읽게 포맷(초→ms 보정).
- 빈 결과/빈 파일 상태 분리. 카피를 catch-all 에 맞게 갱신("전용 메뉴 없는 모든 파일").

- `/api/logs` 는 1회 페치 후 인메모리(3.4MB)에서 즉시 필터/페이지 → **서버 왕복 0**.
